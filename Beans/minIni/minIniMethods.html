Component minIni
minIni is a programmer’s library to read and write 'ini' files in embedded systems.
Component Level: High
Category: SW-User Components
Methods:
(Methods are user-callable functions/subroutines intended for the component functions control. Please see the Embedded Components page for more information.)
  • ini_getbool - Return a boolean key. A true boolean is found if one of the following is matched:
    - A string starting with 'y' or 'Y'
    - A string starting with 't' or 'T'
    - A string starting with '1'
    A false boolean is found if one of the following is matched:
    - A string starting with 'n' or 'N'
    - A string starting with 'f' or 'F'
    - A string starting with '0'

      ANSIC prototype: int ini_getbool(const mTCHAR *Section, const mTCHAR *Key, int DefValue, const mTCHAR *Filename)
    • Section: Pointer to char - The name of the section to search for
    • Key: Pointer to char - The name of the entry to find the value of
    • DefValue:int - Default value in the event of a failed read; it should be zero (0) or one (1).
    • Filename: Pointer to char - The name and full path of the .ini file to read from
    • Return value:int - The true/false flag as interpreted at Key

  • ini_getl - Return the long integral value for a given section and key.

      ANSIC prototype: long ini_getl(const mTCHAR *Section, const mTCHAR *Key, long DefValue, const mTCHAR *Filename)
    • Section: Pointer to byte - The name of the section to search for
    • Key: Pointer to char - Pointer to the name of the entry to find the value of
    • DefValue:long - Default value in the event of a failed read
    • Filename: Pointer to char - The name and full path of the .ini file to read from
    • Return value:long - The value located at Key

  • ini_gets - Return the string for a given section and key.

      ANSIC prototype: int ini_gets(const mTCHAR *Section, const mTCHAR *Key, const mTCHAR *DefValue, nTCHAR *Buffer, int BufferSize, const mTCHAR *Filename)
    • Section: Pointer to byte - The name of the section to search for
    • Key: Pointer to char - Pointer to the name of the entry to find the value of
    • DefValue: Pointer to char - Pointer to the default value in the event of a failed read
    • Buffer: Pointer to byte - A pointer to the buffer to copy into
    • BufferSize:int - The maximum number of characters to copy
    • Filename: Pointer to char - The name and full path of the .ini file to read from
    • Return value:int - The number of characters copied into the supplied buffer

  • ini_getsection - Return the key inside a section

      ANSIC prototype: int ini_getsection(int idx, mTCHAR *Buffer, int BufferSize, const mTCHAR *Filename)
    • idx:int - The zero-based sequence number of the section to return
    • Buffer: Pointer to char - A pointer to the buffer to copy into
    • BufferSize:int - The maximum number of characters to copy
    • Filename: Pointer to char - The name and full path of the .ini file to read from
    • Return value:int - The number of characters copied into the supplied buffer

  • ini_getkey - Return the key inside a section

      ANSIC prototype: int ini_getkey(const mTCHAR *Section, int idx, mTCHAR *Buffer, int BufferSize, const mTCHAR *Filename)
    • Section: Pointer to byte - The name of the section to browse through, or NULL to browse through the keys outside any section
    • idx:int - The zero-based sequence number of the key to return
    • Buffer: Pointer to char - A pointer to the buffer to copy into
    • BufferSize:int - The maximum number of characters to copy
    • Filename: Pointer to char - The name and full path of the .ini file to read from
    • Return value:int - The number of characters copied into the supplied buffer


Note: Some methods can be implemented as macros.
