Component MC13192
Device Driver for the Freescale MC13192 Transceiver
Component Level: Low
Category: CPU External Devices-Communication
Properties:
(Properties are parameters of the component. Please see the Embedded Components page for more information.)
  • Component name - Name of the component.
  • Initialize in Init - If the transceiver shall be initialized during PE_low_level_init()
  • Wait - System wait routines
  • HW Interface - Defines the hardware interface.
    • Transceiver - This defines various signal settings.
      • Transceiver Type - Specifies the transceiver type. Select MC1320x in case you are using a separate transceiver chip. Using the MC1320x (external) transceiver then typically Single Port Mode should be enabled.

        There are 2 options:
        • MC1320x: Separate chip as found on the RF daughter card.
        • MC1321x: SiP version of the transceiver.

      • Single Port Mode - This corresponds to bit 12 (RF_switch_mode) of Control_B register (address 0x07). yes = Single Port Mode selected where RF switch is active and RFIN_M and RFIN_P and bidirectional signals. no = Dual Port Mode selected where RFIN_M and RFIN_P are inputs only and PAO_P and PAO_N are separate outputs. (This is default operation).
      • CT Bias Enable - If ct_bias_en flag in register 7 (Control_B) is enabled or not. This controls the output state.
    • CE Pin - Pin to select the device
    • RESET Pin - Pin to select the device
    • RTXEN Pin - Pin to select the device
    • ATTN Pin - Pin to select the device
    • SPI - Interface to SPI
      • SW SPI - If enabled, a software SPI is used
        The following items are available only if the group is enabled (the value is "Enabled"):
        • SW SPI - Interface to the software SPI
      • HW SPI - If enabled, a hardware SPI is used
        The following items are available only if the group is enabled (the value is "Enabled"):
        • Baud Rate Mode - Baud rate to be used for a shared SPI channel. Index 0 denotes the initial baud rate.
        • HW SPI - Interface to the software SPI
      • MISO High Impedance - This controls bit 11 (miso_hiz_en) of register 0x07 (Control_B). Enabled is the default. The MISO high impedance enable bit either tristates or drives the MISO pin to a logic low when CE is negated. It is recommended to disable this bit for low power modes. (set it to 'no' in this property).
      • MISO Drive Strength - This selects the output drive strength for the transceiver MISO signal. 0 is the lowest strength.
    • IRQ - This configures the interrupt request signal.
      • Macros - Required macros to configure and access IRQ settings
      • IRQ Pin - Interrupt signal from Transceiver
      • IRQ Pullup - This enables the onboard pullup (nominal 40 Ohm) on the IRQ pin. Otherwise the pin is configured as open drain (external pullup required).
      • IRQ Drive Strength - This selects the output drive strength for the transceiver IRQ signal. 0 is the lowest strength.
    • CLKO - Configures the CLKO signal
      • Enable - If the transceiver shall produce a clock signal on CLKO.
      • Frequency - The transciver provides the ability to trim the crystal oscillator frequency and an output clock with a programmable frequency that can be used to drive another device, such as a microcontroller. The field xtal_trim[7:0], CLKO_Ctl Register 0A, Bits 15-8, alter the capacitive loading to the crystal and affects the oscillator frequency.

        There are 8 options:
        • 16 MHz: 16 MHz
        • 8 MHz: 8 MHz
        • 4 MHz: 4 MHz
        • 2 MHz: 2 MHz
        • 1 MHz: 1 MHz
        • 62.5 kHz: 62.5 kHz
        • 32.786 kHz: 32.786 kHz (default)
        • 16.393 kHz: 16.393 kHz

      • Trim bits - The crystal oscillator capacitor trim bits warps the crystal frequency by approximately -0.25 ppm per bit. 0x7E is the default setting and suggested start point for trimming.
      • Toggle in Doze Mode - If the CLKO shall continue to toggle at the selected rate during Doze mode if CLKO is enabled.
      • Drive Strength - This selects the output drive strength for the transceiver CLKO signal. 0 is the lowest strength.
    • GPIOs - If the General Purpose Input and Output pins are enabled. yes = Single Port Mode selected where RF switch is active and RFIN_M and RFIN_P and bidirectional signals. no = Dual Port Mode selected where RFIN_M and RFIN_P are inputs only and PAO_P and PAO_N are separate outputs. (This is default operation).
      The following items are available only if the group is enabled (the value is "Enabled"):
      • GPIO1-4 High Drive Strength - This selects the output drive strength for the transceiver GPIO1 to GPIO4. 0 is the lowest strength.
      • GPIO5-7 High Drive Strength - This selects the output drive strength for the transceiver GPIO5 to GPIO7. 0 is the lowest strength.
      • Alternative MCU interface - If enabled, GPIO1 and GPIO2 are used as status to the MCU. GPIO1 indicates when the transceiver is out of idle. GPIO2 indicates a valid CRC or CCA result. These signals are required for Freescale’s 802.15.4 MAC software.
      • GPIO1 as input - If the IO pin serves as output pin (otherwise as input). If you do not use the pin, configure it as input.
      • GPIO2 as input - If the IO pin serves as output pin (otherwise as input). If you do not use the pin, configure it as input.
      • GPIO3 as input - If the IO pin serves as output pin (otherwise as input). If you do not use the pin, configure it as input.
      • GPIO4 as input - If the IO pin serves as output pin (otherwise as input). If you do not use the pin, configure it as input.
      • GPIO5 as input - If the IO pin serves as output pin (otherwise as input). If you do not use the pin, configure it as input.
      • GPIO6 as input - If the IO pin serves as output pin (otherwise as input). If you do not use the pin, configure it as input.
      • GPIO7 as input - If the IO pin serves as output pin (otherwise as input). If you do not use the pin, configure it as input.
