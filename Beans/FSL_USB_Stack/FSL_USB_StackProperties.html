Bean FSL_USB_Stack
Freescale USB Stack
Component Level: High
Category: SW-Communication
Properties:
(Properties are parameters of the component. Please see the Embedded Components page for more information.)
  • Component name - Name of the component.
  • Freescale USB Stack Version - Version of the Freescale USB Stack used
  • USB Init - USB peripheral to initialize.
    For ColdFire V1, select 'Init_USB_OTG_COLDFIREV1'.
    For ColdFire V2, select 'Init_USB_OTG_MCF'.
    For HCS08, select 'Init_USB_HCS08'.
    For Kinetis, select 'Init_USB_OTG_VAR0'.
  • Device Class - Device class to be used

    There are 5 options:
    • CDC Device: Communication Device Class
    • CDC Host: Communication Device Host
    • HID Keyboard Device: Human Interface Device Class (Keyboard)
    • HID Mouse Device: Human Interface Device Class (Mouse)
    • MSD Host: Mass Storage Device

  • CDC Device - CDC Device Class Interface
    The following items are available only if the group is enabled (the value is "Enabled"):
    • CDCDevice - Interface to CDC class
  • CDC Host - CDC Host Class Interface
    The following items are available only if the group is enabled (the value is "Enabled"):
    • CDCHost - Interface to CDC class
  • HID Keyboard Device - HID Keyboard Device Class Interface
    The following items are available only if the group is enabled (the value is "Enabled"):
    • HID Keyboard - Interface to HID Keyboard class
  • HID Mouse Device - HID Mouse Device Class Interface
    The following items are available only if the group is enabled (the value is "Enabled"):
    • HID Mouse - Interface to HID Mouse class
  • MSD Host - MSD Device Class Interface
    The following items are available only if the group is enabled (the value is "Enabled"):
    • MSDHost - Interface to MSD class
  • DATA_BUFF_SIZE - DATA_BUFF_SIZE should be greater than or equal to the endpoint buffer size, otherwise there will be data loss. For MC9S08JS16, maximum DATA_BUFF_SIZE supported is 16 Bytes.
  • Initialization - Here the initialization of the USB stack can be configured
    • Use USB Stack Inititalization - This defines the value of the USB_USER_CONFIG_USE_STACK_INIT macro in the usb_user_config.h. If set to yes, it uses the USB inititialization of the USB stack. If set to no, it uses the initialization of the Processor Expert USB Init component.
    • Call Init Method - If set to yes, then the Init() method will be called in PE_low_level_init().
