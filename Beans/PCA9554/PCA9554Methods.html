Component PCA9554
NXP PCA9554 and PCA9554A I2C I/O Expander
Component Level: Low
Category: CPU External Devices-Port I/O
Methods:
(Methods are user-callable functions/subroutines intended for the component functions control. Please see the Embedded Components page for more information.)
  • ReadI2CByte - Reads a byte from the I2C device at the given register address.

      ANSIC prototype: byte ReadI2CByte(byte addr, byte *chp)
    • addr:byte - Device register address
    • chp: Pointer to byte - Pointer to byte where to store the value
    • Return value:byte - Error code

  • WriteI2CByte - Writes a byte to the I2C device at the given register address.

      ANSIC prototype: byte WriteI2CByte(byte addr, byte ch)
    • addr:byte - device register
    • ch:byte - value to write
    • Return value:byte - Error code

  • ReadOutput - Reads a byte from the output register.
    This register reflects the outgoing logic levels of the pins defined as outputs by Register 3.
    Bit values in this register have no effect on pins defined as inputs. Reads from this register
    return the value that is in the flip-flop controlling the output selection, not the actual pin
    value.

      ANSIC prototype: byte ReadOutput(void)
    • Return value:byte - output port value

  • WriteOutput - Writes a byte to the output register.

      ANSIC prototype: void WriteOutput(byte val)
    • val:byte - value

  • GetOutputBit - Returns a bit from the output register. It returns zero if the bit is cleared, and non-zero if the bit is set.

      ANSIC prototype: byte GetOutputBit(byte bitNo)
    • bitNo:byte - Bit number, starting with 0
    • Return value:byte - zero if bit is cleared, non-zero otherwise.

  • SetOutputBit - Sets a bit on the output register.

      ANSIC prototype: void SetOutputBit(byte bitNo)
    • bitNo:byte - Bit number, starting with 0

  • ClearOutputBit - Clears a bit on the output register.

      ANSIC prototype: void ClearOutputBit(byte bitNo)
    • bitNo:byte - Bit number, starting with 0

  • NegOutputBit - Negates a bit on the output register.

      ANSIC prototype: void NegOutputBit(byte bitNo)
    • bitNo:byte - Bit number, starting with 0

  • ReadInput - Reads a byte from the input register.
    This register is a read-only port. It reflects the incoming logic levels of the pins, regardless
    of whether the pin is defined as an input or an output by Register 3. Writes to this register
    have no effect. The default is determined by the externally applied logic level, normally ‘1’ when no
    external signal externally applied because of the internal pull-up resistors.

      ANSIC prototype: byte ReadInput(void)
    • Return value:byte - output port value

  • ReadInputBit - Reads a bit from the input register.

      ANSIC prototype: byte ReadInputBit(byte bitNo)
    • bitNo:byte - Bit number, starting with 0
    • Return value:byte - zero if bit is cleared, non-zero otherwise.

  • ReadPolarityInversion - Reads the polarity inversion register.
    This register allows the user to invert the polarity of the Input Port register data. If a bit in
    this register is set (written with ‘1’), the corresponding Input Port data is inverted. If a bit in
    this register is cleared (written with a ‘0’), the Input Port data polarity is retained.

      ANSIC prototype: byte ReadPolarityInversion(void)
    • Return value:byte - Polarity inversion register value.

  • WritePolarityInversion - Writes the polarity inversion register.
    This register allows the user to invert the polarity of the Input Port register data. If a bit in
    this register is set (written with ‘1’), the corresponding Input Port data is inverted. If a bit in
    this register is cleared (written with a ‘0’), the Input Port data polarity is retained.

      ANSIC prototype: void WritePolarityInversion(byte val)
    • val:byte - register value

  • ReadConfig - Reads the polarity inversion register.
    This register configures the directions of the I/O pins. If a bit in this register is set, the
    corresponding port pin is enabled as an input with high-impedance output driver. If a bit in
    this register is cleared, the corresponding port pin is enabled as an output. At reset, the
    I/Os are configured as inputs with a weak pull-up to VDD.

      ANSIC prototype: byte ReadConfig(void)
    • Return value:byte - Configuration register value

  • WriteConfig - Writes the configuration register.
    This register configures the directions of the I/O pins. If a bit in this register is set, the
    corresponding port pin is enabled as an input with high-impedance output driver. If a bit in
    this register is cleared, the corresponding port pin is enabled as an output. At reset, the
    I/Os are configured as inputs with a weak pull-up to VDD.

      ANSIC prototype: void WriteConfig(byte val)
    • val:byte - register value

  • SetAddress - Sets the I2C address of the device to communicate with. The address will be stored in a global variable in the component..Disable this method if you do not need it.

      ANSIC prototype: void SetAddress(byte addr)
    • addr:byte - Device address (to be added to the base address), i.e. starts with zero.

  • GetAddress - Returns the currently used device address used. Note that the address is without the base address (it starts with zero).

      ANSIC prototype: byte GetAddress(void)
    • Return value:byte - Error code


Note: Some methods can be implemented as macros.
