Typical Usage:
(Examples of a typical usage of the component in user code. For more information please see the page Component Code Typical Usage.)

Add your events in the component properties, e.g. 
INIT /* Example event */
SW2_PRESSED      /* SW2 pressed for a short time */
SW3_PRESSED      /* SW3 pressed for a short time */
This will create in the header file the events (with the module prefix in front of the names): 
/* List of events */
#define EVNT1_INIT             /* Example event */                0
#define EVNT1_SW2_PRESSED      /* SW2 pressed for a short time */ 1
#define EVNT1_SW3_PRESSED      /* SW3 pressed for a short time */ 2
You can clear, set or check the events from your code, e.g. to set an event in a keyboard interrupt routine: 
void OnKBI(void) {
  EVNT1_SetEvent(EVNT1_SW2_PRESSED);
}
In your application you can check if any events are pending: 
  for(;;) {
    if (EVNT1_EventsPending()) { /* this is an optional check */
      EVNT1_HandleEvent(); /* will call event routine for events */
    }
  }
If there are any events, then it will call the event method (generated in Events.c). In there you can handle the event: 
/* Events.c */
void EVNT1_AppHandleEvent(byte event)
{
  APP_HandleEvent(event);
}
In above implementation, it calls a handler in our application, e.g.: 
void APP_HandleEvent(byte event) {
  switch(event) {
    case EVNT1_SW2_PRESSED:
      LED1_Neg();
      break;
    case EVNT1_SW3_PRESSED:
      LED2_Neg();
      break;
  }
}
