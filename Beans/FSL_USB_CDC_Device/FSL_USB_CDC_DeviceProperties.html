Bean FSL_USB_CDC_Device
Freescale USB Stack (CDC)
Component Level: High
Category: SW-Communication
Properties:
(Properties are parameters of the component. Please see the Embedded Components page for more information.)
  • Component name - Name of the component.
  • CPU - Device to be used.

    There are 16 options:
    • MC9S08JS16: MC9S08JS16
    • MC9S08JM16: MC9S08JM16
    • MC9S08JM60: MC9S08JM60
    • MC9S08MM128: MC9S08MM128
    • MCF51JM64: ColdFire V1 MCF51JM64
    • MCF51JM128: ColdFire V1 MCF51JM128
    • MCF51MM256: ColdFire V1 MCF51MM256
    • MCF52259: ColdFire V2 MCF52259
    • Kinetis KL25: Kinetis L2K family (e.g. KL25Z)
    • Kinetis KL46Z48: TWR-KL46Z48M, FRDM-KL46Z48M
    • Kinetis K20D50: Kinetis K20D50 (e.g. FRDM-K20D50M)
    • Kinetis K20D72: Kinetis K20 72 MHz (K20DX)
    • Kinetis K21D50: Kinetis K21D50 (e.g. TWR-K21D50M)
    • Kinetis K40: Kinetis K40
    • Kinetis K60: Kinetis K60
    • Kinetis K70: Kinetis K70

  • CDC Settings - Settings for USB CDC class. The .inf file will be generated into the Documentation folder.
    The following items are available only if the group is enabled (the value is "Enabled"):
    • .inf ClassGuid - This defines the ClassGuid in the cdc.inf file. You can make your own using http://www.guidgenerator.com/
    • .inf VID - Value to be used in the cdc.inf for the VID (Vendor ID) entry. This value is used as well in usb_descriptor.c.
    • .inf PID - Name to be used in the cdc.inf for the PID (Product ID) entry. This value is used as well in usb_descriptor.c.
    • .inf PRVDR - Name to be used in the cdc.inf for the PRVDR (Provider) entry.
    • .inf MFGNAME - Name to be used in the cdc.inf for the MFGNAME entry.
    • .inf DESCRIPTION - Name to be used in the cdc.inf for the DESCRIPTION entry.
    • .inf SERVICE - Name to be used in the cdc.inf for the SERVICE entry.
    • Bus reported device - This name shows up on the USB bus before drivers are installed. This string is used in usb_descriptor.c.
    • Bus reported vendor - This name shows up on the USB bus before drivers are installed. This string is used in usb_descriptor.c.
  • Send Buffer - Interface to ring for sending data.
  • Receive Buffer - Interface to ring for sending data.
  • Use Timeout - If enabled, the application task implements a timeout so it does not block.
    The following items are available only if the group is enabled (the value is "Enabled"):
    • Timeout - Interface to timeout module. See the timeout documentation how to use it.
    • App Task Timeout (ms) - The App_Task() is sending data to the CDC. However, if there is no terminal open on the host, then the App_Task() function could be stalled. A timeout can be specified here. Using a zero value disables the timeout.
