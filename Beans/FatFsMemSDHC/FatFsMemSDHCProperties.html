Component FatFsMemSDHC
Memory Implementation for FatFS using SDHC_LDD
Component Level: Low
Category: SW-User Components
Properties:
(Properties are parameters of the component. Please see the Embedded Components page for more information.)
  • Component name - Name of the component.
  • SDHC - Interface to SHDC
  • Speed Mode - Defines the bus speed modes (SPI clock). For compatibility to older cards, the card needs to be initialized with bus clock less than 400 kHz, which then can be increased up to 12 MHz. Certain high speed cards support an even higher speed. The values in the group indicate indicies to the speed values as set up in the SDHC component (list of values).
    • Speed Index Slow - Index into speed table. This has to be less than 400 kHz to initialize the card.
    • Speed Index Normal - Index into speed table. Typically this is up to 12 MHz. This speed is used after initialization of the card (if not even a high speed mode is supported by the card).
    • Speed Index Fast - Index into speed table. Some cards support fast mode higher than 12 MHz.
  • Card Detect Pin - Configures an optional card detect pini
    The following items are available only if the group is enabled (the value is "Enabled"):
    • CD - Interface to card detect pin. Note that many cards need a pull-up or even pull-down resistor added.
    • Low Active - Set to 'yes' if a card inserted will be LOW logical level, otherwise set it to 'no'
    • Pull Code - Here optional pull-up/pull down code can be specified. This code is added to the Initialization code.
      The following items are available only if the group is enabled (the value is "Enabled"):
      • Include Code - Here a header file include can be specified which is include just before Init()
      • Pull Code - Here user pull-up/pull-down code can be added. This code is inserted into the Init code.
  • Write Protect Pin - Optional pin to detect if the card is write protected.
    The following items are available only if the group is enabled (the value is "Enabled"):
    • WP pin - Interface to write protect pin
    • Low Active - If pin is low active (logical low level means write protection is on)
  • Timeout - Interface to timeout functionality
